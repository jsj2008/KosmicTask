NDScriptData

Class for representing script data OSAID

Conforms to: NSCopying, NSCoding
Superclass: NSObject
Declared In: NDScriptData.h

Discussion

NDScriptData is the base class for NDScriptHandler and NDSccriptContext. Instances of NDScriptData can represent script data.

initWithContentsOfFile:componentInstance:loadedFrom:

Initialize a NDSciptData 

- (id)initWithContentsOfFile:(NSString *)path 
        componentInstance:(NDComponentInstance *)componentInstance 
        loadedFrom:(enum LoadedFrom*)loadedFrom;
Parameters
path
The path of the script file.
componentInstance
The component instance to use, a value of nil to use the default AppleScript context.
loadedFrom
A pointer to a enum LoadedFrom which on return contains on of the three values LoadedFromDataFork, LoadedFromResourceFork, LoadedFromTextFile, can be set to NULL.
Return Value

A initialized NDScriptData, or nil of initialization fails.

Discussion

Intialises the reciever with the contents of the file at the path path, if script data can not be found within data fork the any script data within the resource fork, if that fails then an attempt will be made to load the file as if it was an uncomplied text file. On return the value pointered to by loadedFrom will contain one of the following values;

Compiled script data was found in the data fork and used to initialize the reciever.
Compiled script data was found in the resource fork and used to initialize the reciever.
Source code was found in the data fork that could be compiled was used.

isEqualToScriptData:

Test if two script datas are equal. 

- (BOOL)isEqualToScriptData:(NDScriptData *)scriptData;
Parameters
scriptData
The script data to compere the reciever with.
Return Value

This method returns YES if the reciever and scriptData are equal; otherwise, it returns NO.

Discussion

Compares the receiving script data object to scriptData. If the contents of scriptData is equal to the contents of the receiver, this method returns YES. If not, it returns NO. Two data objects are equal if they both have the same component instance and there script ids (OSAID) are equal.

writeToFile:atomically:

Write script data to a file. 

- (BOOL)writeToFile:(NSString *)path atomically:(BOOL)atomically;
Parameters
path
The file path.
atomically
Write to a backup file first.
Return Value

This method returns YES if the operation succeeds; otherwise, it returns NO.

Discussion

Writes the script data in the receiver to the file specified by path. If atomically is YES, the script data is written to a backup file, and then, assuming no errors occur, the backup file is renamed to the specified filename. Otherwise, the script data is written directly to the specified file.

writeToURL:atomically:

Write script data to a file. 

- (BOOL)writeToURL:(NSURL *)URL atomically:(BOOL)atomically;
Parameters
URL
The file URL.
atomically
Write to a backup file first.
Return Value

This method returns YES if the operation succeeds; otherwise, it returns NO.

Discussion

Writes the script data in the receiver to the location specified by URL. If atomically is YES, the script data is written to a backup location, and then, assuming no errors occur, the backup location is renamed to the specified name. Otherwise, the script data is written directly to the specified location. atomically is ignored if URL is not of a type the supports atomic writes.
